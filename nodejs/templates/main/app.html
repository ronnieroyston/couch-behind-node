My Web Application

This page displays files that have been uploaded to CouchDB by individual users. Note that the term bucket is used here but in CouchDB terminology files are stored as attachments to documents.

User Management

The process of email link authentication begins with collecting a users email address and POST'ing it to the Node.js server which then generates a random ID and stores it along with the users information in CouchDB. Once the user recieves the email and clicks the link a GET request is sent to Node.js that includes the random ID which if found in CouchDB results in a JSON Web Token being minted and returned to the client in an HTTP cookie. From then on the users ID is sent to the server in each HTTP call.

Server Assembled in Milliseconds

All HTML files are assembled by the Node.js server. The homepage is assembled and stored as index.html prior to starting the HTTP server. This /app page is assembled from disparate files on the fly by the server in response to HTTP request to the /app path. In other words there is no app.html file stored on the server. Instead, there is a header.html, main.html, and footer.html file. Both approaches are illustrated but pre-built HTML is always fastest and most performant.
